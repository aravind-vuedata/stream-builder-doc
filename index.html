Stream Builder Tech Doc

Goal

Enable non-engineering users to create pages, place widgets, and customize each widget from admin tools, while mobile apps render UI from server JSON without app releases.

Scope

  • Page composition and ordering
  • Widget configuration and customization
  • Contract versioning and backward compatibility
  • Atom-level component contract standardization

High-Level Flow

  1. Creator builds page in Stream Builder CMS.
  1. CMS stores page config JSON.
  1. Mobile app requests page by pageId.
  1. Backend returns typed JSON contract.
  1. Client renderer maps widget.type to native component.
  1. Client executes actions/events from contract.

Widget System

  • Widgets allow non-engineering users to assemble pages in CMS while mobile apps render them dynamically using atomic components.

A page is composed of three primary containers:

  • Header → Top navigation or branding
  • Body → Main content widgets
  • Footer → Bottom Tab or information

Page Contract
FieldTypeRequiredDescription
page_idstringYesUnique page identifier
